Build a full-stack Levy Collection Practice Management Platform (GhostPractice Replacement) with MVP focus, prioritizing document exporting and core matter management first, then dashboards and reporting.
🎯 PRIORITY ORDER (CRITICAL)
Document Exporting (PDF generation) – Generate all legal documents (LOD, S129, Summons, Judgment, Writ, Rule 46, Joinder, Payment Arrangement) with automatic merge fields, interest-to-date calculation, and one-click download. Bulk document generation for multiple matters.

Matter Management – Create matters with auto‑reference (BAM‑2026‑0001), link debtor + scheme + unit + managing agent, pipeline stages (LOD → S129 → Summons → Judgment → Writ → Rule46 → Sale → Closed), stage advancement with validation.

Debtor & Scheme Management – Full debtor profiles (name, ID, phone, WhatsApp, email, address), scheme register, managing agent registry.

Financial Tracking – Capital arrears, interest (prescribed rate engine), legal costs, payment receipting, running balance per matter.

Diary / Task Automation – Auto‑generate tasks when a matter changes stage (e.g., “Await S129 response” due in 10 business days). Manual tasks, due date tracking, “My Tasks” view.

Dashboard & Reporting – Pipeline funnel, total outstanding per stage, aged debtors report, per‑agent/scheme collections, matters requiring attention.

Authentication & RBAC – Roles: Admin, Attorney, Collector, Agent Viewer (read‑only portal). JWT + bcrypt.

Agent Self‑Service Portal (read‑only) – Agents can view their schemes, matters, balances, documents, timeline (no write access).

Reporting Layer – CSV/Excel export for all reports.

MVP mantra: Document export works perfectly → matter tracking is bulletproof → financials are correct → everything else is polish.

Architecture choices (simpler than microservices – MVP speed matters):

Monorepo using Next.js + tRPC → end‑to‑end type safety

PostgreSQL (via Prisma ORM) – single source of truth

Redis (optional) for background job queue (BullMQ) – used for bulk document generation and nightly interest recalculation

Cloudflare R2 (S3‑compatible) – PDF storage

No separate API gateway – Next.js API routes handle auth & routing

Event‑driven only where needed – async doc generation via BullMQ

Production‑ready structure but stripped for rapid MVP delivery.

📄 Document Exporting – Detailed Requirements (MVP #1)
Supported templates (MVP):

Letter of Demand (LOD)

Section 129 Notice

Combined Summons (Magistrate’s Court)

Default Judgment application

Writ of Execution

Rule 46(1) Notice

Section 15 STSMA Joinder Notice

Payment Arrangement letter

Monthly Statement to Managing Agent

Merge fields – all templates must auto‑populate from matter/debtor/scheme data:

{debtor_name}, {debtor_address}, {debtor_id}

{scheme_name}, {unit_number}, {levy_arrears}

{capital_arrears}, {interest_to_date}, {legal_costs}, {total_due}

{interest_rate}, {interest_per_day}

{reference}, {stage}, {lod_date}, {s129_date}

{court_name}, {case_number} (for summons/writ)

Interest calculation:

Must compute interest from interest_from_date (or lodDate) to current date using prescribed rate (or matter‑specific override).

Display interest breakdown in document: principal, rate, days, amount.

Use multi‑period logic if prescribed rate changed during period.

Bulk document generation:

UI: select multiple matters (filter by stage, scheme, agent) and a template.

Submit async job via BullMQ.

Show job status (pending → processing → completed/failed).

When complete, list generated PDFs in a table with download links.

Support CSV export of generation log.

Storage & retrieval:

Each generated PDF stored in Cloudflare R2 with path: matters/{matterId}/{docType}_{timestamp}.pdf.

Record in Document table with file URL, generation timestamp, user.

Frontend: download via signed URL (expires in 1 hour).

Export formats:

Primary: PDF (print‑ready, A4, proper margins, court‑style formatting).

Also support DOCX download (original template filled) for editing.

MVP must pass this test: A collector can select 50 matters in “LOD” stage, click “Generate Letter of Demand”, walk away, come back in 2 minutes, and download 50 PDFs each with correct interest calculated to that day.
